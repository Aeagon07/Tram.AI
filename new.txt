# 1️⃣ Architecture Strategy (Very Important)

We will use:

* Flutter (Stable)
* Dart
* Material 3 (customized)
* Riverpod or Provider (state management)
* go_router (navigation)

# 3️⃣ Core Design System (Flutter Version)

---

## 🎨 App Theme Setup

Create:

`app_theme.dart`

Use:

* ColorScheme.fromSeed
* Custom HSL-like color harmony
* Dark-first design

Define:

* Primary: Urban Blue
* Eco: Green
* Alert: Red
* Glass surface color

Enable smooth theme switching using:

```dart
AnimatedTheme(
  data: theme,
  duration: Duration(milliseconds: 300),
  child: ...
)
```

# 6️⃣ Animation Strategy

Use:

* flutter_animate
* Hero animations
* AnimatedSwitcher
* AnimatedContainer

Keep animations:

* 200–400ms
* Smooth curves (Curves.easeOut)

Avoid over-animation.

---

# 7️⃣ Mock Data Layer

Create:

`mock_routes.dart`

Example:

```dart
final routes = [
  RouteModel(
    mode: "Metro",
    time: "28 min",
    cost: 40,
    congestion: "Low",
    ecoScore: 92,
    recommended: true,
  )
];
```

UI reads from mock only.

No backend yet.

# 8️⃣ Performance Rules

* Avoid heavy shadows
* Use const constructors
* Use ListView.builder
* Avoid rebuilding entire page
* Use state management properly

Target 60 FPS on Samsung S23.

# 🔥 Final Advice

Switching to Flutter is not wrong.

But only do it if:

* You’re early stage
* You want full control over UI
* You’re confident in Dart

If you’re under time pressure,
Next.js + Capacitor is faster.

If you want premium native UI polish,
Flutter is stronger.
